### 3. macOS support claim is absent, but Mac users WILL try it — and the cache path is wrong
- **Where:** `/home/gas6amus/Documents/be-my-geminis/internal/cache/cache.go:103-111`
- **Why it matters:** README:10 says "Tested on Linux"; cache.go falls back through `BMG_CACHE_DIR` → `$XDG_RUNTIME_DIR` → `/tmp/bmg-cache`. On macOS, `$XDG_RUNTIME_DIR` is unset by default, so a Mac user lands on `/tmp/bmg-cache` — which is world-writable, **shared across users**, and not auto-cleaned (security side-channel already in Batch 1 cache-mode finding, but the Mac case makes it the default, not the edge case). The canonical macOS location is `~/Library/Caches/bmg` via `os.UserCacheDir()`. Also: `~/.claude/settings.json` and `~/.claude.json` happen to be correct on macOS (CC writes there directly, not `~/Library/Application Support/...`) — verified consistent with Claude Code's documented layout.
- **Fix:** Add a fourth tier between XDG_RUNTIME_DIR and `/tmp`:
  ```go
  if d, err := os.UserCacheDir(); err == nil {
      return filepath.Join(d, "bmg"), nil
  }
  ```
  `os.UserCacheDir()` returns `~/Library/Caches` on macOS, `$XDG_CACHE_HOME` or `~/.cache` on Linux, `%LocalAppData%` on Windows. This single change makes the cache safe and discoverable on all three platforms and lets you drop the `/tmp/bmg-cache` fallback (or keep it as last-ditch). Update doc comment in `cache.go` to match.
